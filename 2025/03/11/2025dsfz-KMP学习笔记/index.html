前言：这把高端局

关于KMP

  • 时间复杂度为 O(n+m)O(n+m)O(n+m) 的优秀字符串查找算法。
  • 适用于在句子/文章中查找一段文字（词语）。

KMP实现

关于共同前后缀数组（PMT）

  • 说人话就是 nextnextnext 数组。

  • 是什么？

    • nextinext_inexti​ 表示下标从 111 到 iii 的子串（后文中叫做 OrginOrginOrgin 串）中既是 OrginOrginOrgin 的前缀串又是 OrginOrginOrgin 的后缀串的字符串的最长长度。
    • 举例说明：开始时候的串为：abcab
      • 那 next5=2next_5=2next5​=2 ，因为存在串 ababab 既是 OrginOrginOrgin 的前缀又是 OrginOrginOrgin 的后缀，而且在所有满足条件的串中，它是最长的。所以把 next5next_5next5​ 设为它的长度—— 222.

  • 构建 PMTPMTPMT 的详细步骤

    • 初始化：设模式串为 PPP，长度为 mmm，创建一个长度为 mmm 的数组 pmtpmtpmt 来存储部分匹配表(相当于 nextnextnext 数组)的值。初始化 pmt[0]=0pmt[0] = 0pmt[0]=0，因为模式串的第一个字符的前缀子串只有一个字符，不存在相等的前后缀。
    • 遍历模式串：从模式串的第二个字符开始遍历，即 i=1i = 1i=1 到 i=m−1i = m - 1i=m−1。对于每个位置 iii，设两个指针，一个指针 j 指向当前前缀子串的最长相等前后缀的下一个位置，初始时 j=pmt[i−1]j = pmt[i - 1]j=pmt[i−1]。
      -比较字符：比较 P[i]P[i]P[i] 和 P[j]P[j]P[j]。如果 P[i]P[i]P[i] 等于 P[j]P[j]P[j]，说明找到了更长的相等前后缀，此时 pmt[i]=j+1pmt[i] = j + 1pmt[i]=j+1，然后将 jjj 后移一位，即 j=j+1j = j + 1j=j+1，继续下一个位置的比较。
    • 不相等情况：如果 P[i]P[i]P[i] 不等于 P[j]P[j]P[j]，且 j>0j > 0j>0，则将 jjj 更新为 pmt[j−1]pmt[j - 1]pmt[j−1]，继续比较 P[i]P[i]P[i] 和 P[j]P[j]P[j]。这是因为当 P[i]P[i]P[i] 和 P[j]P[j]P[j] 不相等时，需要回溯到 jjj 位置之前的最长相等前后缀的下一个位置，继续进行比较。
    • 最终结果：当遍历完整个模式串后，pmt(next)pmt(next)pmt(next) 数组中存储的就是模式串的部分匹配表。

  • 示例

    • 以模式串 “ababaca” 为例：
    • 首先初始化 pmt[0]=0pmt[0] = 0pmt[0]=0。
      • 对于 i=1i = 1i=1，即字符 “b”，其前缀子串 “a” 不存在相等前后缀，所以 pmt[1]=0pmt[1] = 0pmt[1]=0。
      • 对于 i=2i = 2i=2，字符 “a”，前缀子串 “ab” 也不存在相等前后缀，pmt[2]=0pmt[2] = 0pmt[2]=0。
      • 对于 i=3i = 3i=3，字符 “b”，此时前缀子串 “aba”，发现前缀 “a” 和后缀 “a” 相等，所以 pmt[3]=1pmt[3] = 1pmt[3]=1。
      • 对于 i=4i = 4i=4，字符 “a”，前缀子串 “abab”，最长相等前后缀为 “ab”，所以 pmt[4]=2pmt[4] = 2pmt[4]=2。
      • 对于 i=5i = 5i=5，字符 “c”，先看 j=pmt[4]=2j = pmt[4] = 2j=pmt[4]=2，比较 “c” 和 “b” 不相等，再将 jjj 更新为 pmt[2−1]=0pmt[2 - 1] = 0pmt[2−1]=0，此时 “c” 和 “a” 也不相等，所以 pmt[5]=0pmt[5] = 0pmt[5]=0。
      • 对于 i=6i = 6i=6，字符 “a”，先看 j=pmt[5]=0j = pmt[5] = 0j=pmt[5]=0，比较 “a” 和 “a” 相等，所以 pmt[6]=1pmt[6] = 1pmt[6]=1。
      • 最终得到的 PMT 数组为 [0, 0, 0, 1, 2, 0, 1]。
